# Phase 7.5 — Test Backlog Design

## Goal

Add ~27-30 fast tests to the Playwright suite, covering user-facing flows deferred from Phase 7. No new slow tests — everything is covered via route intercepts or DB seeding.

## Approach

Extend existing fixtures (`freshAnonPage`, `mockApi.checkoutStatus`, `supabaseAdmin.setSubscriptionState`). New test files go flat into `tests/fast/`. One new page object for the landing page. No structural changes to config, global setup, or teardown.

---

## Section 1: New Fixtures

### `freshAnonPage`

Identical to `anonPage` but without `dismissCookieConsent()`. Used exclusively by cookie consent tests. Keeps `hideDevOverlay` (harmless, prevents flaky clicks).

```ts
freshAnonPage: async ({ browser }, use) => {
  const context = await browser.newContext();
  await hideDevOverlay(context);
  const page = await context.newPage();
  await use(page);
  await context.close();
},
```

### `mockApi.checkoutStatus`

Route-intercepts `/api/stripe/checkout/status` to return a canned response. Used by checkout success page tests.

```ts
checkoutStatus: async (page: Page, response: { plan: string }) => {
  await page.route("**/api/stripe/checkout/status", (route) =>
    route.fulfill({
      status: 200,
      contentType: "application/json",
      body: JSON.stringify(response),
    })
  );
},
```

### `supabaseAdmin.setSubscriptionState` and `resetSubscription`

The free test user has no `subscriptions` row by default (free tier by absence). `setSubscriptionState` upserts a row; `resetSubscription` deletes it entirely to restore free tier.

```ts
// In supabase-admin.ts
async function setSubscriptionState(
  userId: string,
  overrides: {
    status?: string;           // 'active' | 'past_due' | 'cancelled'
    cancelAtPeriodEnd?: boolean;
    currentPeriodEnd?: string;  // ISO date
    gracePeriodEnd?: string;    // ISO date
  }
) {
  // Upsert into subscriptions table with plan='pro' + overrides
}

async function deleteSubscription(userId: string) {
  // Delete from subscriptions where user_id = userId
}
```

Exposed via the `supabaseAdmin` fixture:

```ts
supabaseAdmin: {
  // existing
  setUsageCount, resetUsage, getFreeUserId, getProUserId,
  // new
  setSubscriptionState: (userId, overrides) => setSubscriptionState(userId, overrides),
  resetSubscription: (userId) => deleteSubscription(userId),
}
```

## Section 2: New Page Object

### `playwright/pages/landing.page.ts`

Lightweight page object for landing page sections. Keeps `landing-sections.spec.ts` clean and selectors reusable.

```ts
export class LandingPage {
  constructor(private page: Page) {}

  // Nav
  get nav() { return this.page.locator('nav'); }
  get signInLink() { return this.nav.getByRole('link', { name: /sign in/i }); }
  get getStartedLink() { return this.nav.getByRole('link', { name: /get started/i }); }
  get logoLink() { return this.nav.getByRole('link', { name: /plexease home/i }); }

  // How It Works
  get howItWorksSection() { ... }
  get howItWorksSteps() { ... }

  // Tools
  get toolsSection() { ... }
  get nugetCard() { ... }

  // Pricing
  get pricingSection() { ... }
  get pricingToggle() { ... }
  get freeCard() { ... }
  get proCard() { ... }

  // Attribution
  get attribution() { ... }

  // Footer
  get footer() { return this.page.locator('footer'); }
  get footerTermsLink() { ... }
  get footerPrivacyLink() { ... }
  get manageCookiesButton() { ... }

  async goto() { await this.page.goto('/'); }
}
```

## Section 3: New Test Files (Fast)

### `tests/fast/landing-sections.spec.ts` (~8-10 tests)

Uses `anonPage` + `LandingPage` page object.

| Test | Assertion |
|------|-----------|
| Nav sign-in link | href → `/login` |
| Nav get-started link | href → `/signup` |
| Nav logo link | href → `/` |
| How It Works renders 3 steps | 3 step elements visible |

| Footer has 3 columns | Brand, Product, Legal headings |
| Footer legal links | Terms → `/terms`, Privacy → `/privacy` |

### `tests/fast/cookie-consent.spec.ts` (~5 tests)

Uses `freshAnonPage` exclusively.

| Test | Assertion |
|------|-----------|
| Banner appears on first visit | Cookie consent dialog visible at `/` |
| Banner hidden on /privacy | Navigate to `/privacy`, dialog not visible |
| Accept dismisses + persists | Click Accept, banner gone, reload, still gone |
| Reject dismisses + persists | Click Reject, banner gone, reload, still gone |
| Manage Cookies resets consent | Use `anonPage` (consent pre-set), click "Manage Cookies" in footer, banner reappears |

Note: the "Manage Cookies" test uses `anonPage` (which has consent pre-accepted) to test the reset flow.

### `tests/fast/auth-pages.spec.ts` (~4-5 tests)

Uses `anonPage`.

| Test | Assertion |
|------|-----------|
| Check-email page renders | Heading "Check your email", "Back to sign in" link |
| Forgot-password page renders | Email input, submit button, "Sign in" link |
| Forgot-password empty submit | Shows validation error |
| Reset-password page renders | Heading "Set new password", password input visible |

Reset-password: without a valid token the form may show an error state — soft-assert that the page renders something (heading or error), proving the route works.

### `tests/fast/upgrade-page.spec.ts` (~3-4 tests)

Uses `freeUserPage` + existing `UpgradePage` page object.

| Test | Assertion |
|------|-----------|
| Heading visible | "Upgrade" or equivalent heading |
| Monthly pricing shown by default | Pro price visible |
| Toggle to annual | Price changes, "Save" text appears |
| Subscribe button visible | CTA button present |

### `tests/fast/billing-banners.spec.ts` (~4 tests)

Uses `freeUserPage` + `supabaseAdmin`. Runs **serially** with `afterEach` cleanup.

| Test | Assertion |
|------|-----------|
| Cancelled banner | Seed `cancel_at_period_end: true, status: 'active'` → amber banner, "Resubscribe" button |
| Grace banner | Seed `grace_period_end: <tomorrow>` → red banner "expires tomorrow", "Resubscribe" button |
| Past due banner | Seed `status: 'past_due'` → red banner "Payment failed", "Update payment method" button |
| No banner (baseline) | No subscription row → no banners visible |

Each test seeds via `supabaseAdmin.setSubscriptionState`, asserts, then `afterEach` calls `supabaseAdmin.resetSubscription` to restore free tier.

### `tests/fast/checkout-success.spec.ts` (~2 tests)

Uses `freeUserPage` + `mockApi.checkoutStatus`.

| Test | Assertion |
|------|-----------|
| Happy path | Mock `{ plan: "pro" }` → navigate to `/upgrade/success` → redirects to `/dashboard` |
| Loading state | Mock `{ plan: "free" }` → navigate to `/upgrade/success` → spinner visible, "Setting up your Pro account" text |

Skip testing the actual timeout threshold — that's implementation detail. The timeout UI can be tested later if needed.

## Out of Scope

| Item | Reason |
|------|--------|
| SEO meta tag assertions | Brittle, break on copy changes, low regression risk |
| JSON-LD validation | Static content, manually verifiable |
| Sitemap/robots.txt tests | Trivially verifiable, rarely changes |
| 500 error page test | Difficult to trigger reliably; 404 test validates error page pattern |
| Forgot/reset password email flow | Fragile, rate-limited, UI tests cover the important parts |
| Resubscribe integration test | Requires real Stripe subscription lifecycle; button visibility tested in billing banner tests |
| Test file grouping into subdirectories | Premature at ~12 fast files; revisit when count exceeds ~25 |
| MSW migration | Deferred until second API integration is added |
